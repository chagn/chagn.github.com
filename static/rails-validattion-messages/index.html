Rails validattion messages

29 July 2017 on Ruby on Rails, i18n

in zh-TW.yml 

---
zh-TW:  
  errors:
    # 錯誤訊息格式
    # attribute => 欄位名稱 
    # message => 錯誤訊息 
    # count => 對應數量
    # value => 輸入的值
    format: "%{attribute} %{message}"
    messages:
      accepted: 必須是可被接受的
      # validate method => presence
      blank: 不能為空白
      present: 必須是空白
      # validate method => confirmation
      confirmation: 兩次輸入須一致
      empty: 不能留空
      equal_to: 必須等於 %{count}
      even: 必須是偶數
      exclusion: 是被保留的關鍵字
      greater_than: 必須大於 %{count}
      greater_than_or_equal_to: 必須大於或等於 %{count}
      # validate method => inclusion 
      inclusion: "%{value} 沒有包含在列表中"
      invalid: 是無效的
      less_than: 必須小於 %{count}
      less_than_or_equal_to: 必須小於或等於 %{count}
      model_invalid: "校驗失敗: %{errors}"
      not_a_number: 不是數字
      not_an_integer: 必須是整數
      odd: 必須是奇數
      required: 必須存在
      taken: 已經被使用
      too_long:
        one: 過長（最長是一個字）
        other: 過長（最長是 %{count} 個字）
      too_short:
        one: 過短（最短是一個字）
        other: 過短（最短是 %{count} 個字）
      wrong_length:
        one: 字數錯誤 (必須是一個字)
        other: 字數錯誤 (必須是 %{count} 個字)
      other_than: 不可以是 %{count} 個字

%{attribute} %{count} %{value} 均可以使用在model客製化message中.

想法
  1. model.errors.full_messages.join("</br>")
    但如果一個欄位有多個驗證未過, 欄位名稱會重複
  2. model.errors.messages.map{|k,v| "#{k} #{v.join("與")}"}.join("</br>")
    這樣會整理同一個欄位的錯誤訊息
  3. model.errors.messages.values.join("</br>")
    這樣只會顯示錯誤訊息, 適合客製化訊息的時候
參考
  • https://github.com/svenfuchs/rails-i18n/blob/master/rails/locale/zh-TW.yml
    有裝gem 'rails-i18n' 會自動使用這份, 如需要修改可以在config/locales 下新增這隻檔案再自行修改.
